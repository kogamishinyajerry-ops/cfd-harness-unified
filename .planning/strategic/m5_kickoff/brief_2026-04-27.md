## M5 kickoff specification (verbatim from user prompt)

### Goal (this PR · M5)

Ship workbench-side STL import that takes a user-uploaded STL, runs trimesh
sanity checks, scaffolds a minimal OpenFOAM case directory by forking the
LDC template, and routes the user into the existing `/workbench/case/{id}/edit`
flow.

### Non-goals (explicit)

STEP/IGES support · geometry healing · UI patch picking · cfMesh · NL · LLM
agent on case generation.

### Deliverables (≤ ~750 LOC main + ≤ ~300 LOC tests)

- **A. Backend module · `ui/backend/services/geometry_ingest/`**:
  `stl_loader.py` (trimesh-based loader), `health_check.py` (watertight,
  bbox, unit-guess m/mm/in via bbox heuristic, solid count, face count,
  single-shell check), `patch_detector.py` (per-solid name → patch list;
  honest "all defaultFaces" degradation), `__init__.py` (public
  `IngestSTL(bytes) -> IngestReport`).
- **B. Backend module · `ui/backend/services/case_scaffold/`**:
  `template_clone.py` (copy LDC template tree to
  `user_drafts/imported/{case_id}/`), `bc_injector.py` (write
  `triSurface/geometry.stl` + minimal `snappyHexMeshDict` stub referring to
  detected patches; do NOT run sHM yet — that is M7), `manifest_writer.py`
  (emit `case_manifest.yaml` with `source: "imported"`,
  `origin_filename`, `ingest_report_summary`, `created_at`,
  `solver_version_compat`).
- **C. Backend route · `ui/backend/routes/import_geometry.py`**:
  `POST /api/import/stl` (multipart, max 50 MB enforced in route),
  200 → `{ case_id, ingest_report, edit_url }`, 4xx → honest fail with
  which sanity check broke.
- **D. Frontend page · `ui/frontend/src/pages/workbench/ImportPage.tsx`**:
  Drop-zone or file picker, ingest report card (watertight ✓/✗, bbox, unit
  guess, patches, faces), inline help on WARN, "Create case" button → POST
  + redirect to `/workbench/case/{case_id}/edit`.
- **E. Tests**: `test_geometry_ingest.py` + `test_case_scaffold.py` +
  `test_import_route.py`. Cases: 1 happy-path (named-solid sphere), 1
  watertight-fail, 1 named-solid-missing degradation, 1 over-50MB rejection,
  1 scaffold-from-LDC asserts case_manifest schema.
- **F. Dependency**: Add `trimesh>=4.0` to
  `pyproject.toml` `[project.optional-dependencies].workbench` (NOT default).

### Spec-mandated governance gates

1. Pre-implementation scan recorded · then write code
2. Self-test: `pytest ui/backend/tests/services/test_geometry_ingest.py -q` passes locally before PR
3. Codex 3-round review (this PR touches new upload surface + new pip dep · spec wording: "NOT trust-core but adjacent")
4. After Codex APPROVE_WITH_COMMENTS-or-cleaner · land to main with commit trailer
   `DEC-V61-08X · M5 STL Case Import v0` + Strategic-by/Code-by/Archive-by lines

### Locked decisions D1–D8 (cross-cutting, binding M6–M8)

- **D1**: `examples/imports/` inline in repo · ≤ 8 MB total · 3 fixtures
  (`ldc_box.stl <50KB`, `cylinder.stl <100KB`, `naca0012.stl <300KB`) · NO Git LFS.
- **D2**: Geometry persistence — once "Create case" clicked, geometry lives
  at `user_drafts/imported/{case_id}/` permanently until user deletes case ·
  staging area for unconfirmed uploads at `_staging/` auto-cleans after 30
  days · M5 may stub the TTL job (cron entry + empty function).
- **D3**: `source_origin = "imported_user"` written into
  `case_manifest.yaml` AND propagated into RunHistory schema. **TrustGate
  MUST hard-cap imported cases at PASS_WITH_DISCLAIMER** (no "PASS" possible
  without gold-standard reference) — 1-line guard in TrustGate routing AS
  PART OF M5, not deferred.
- **D4**: M6 `gmsh>=4.11,<4.13` · CI smoke install on ubuntu-latest +
  macos-13 arm64 · macOS arm64 wheel failure documented as known issue with
  conda-forge fallback in install docs · NOT an M6 blocker.
- **D5**: Imported cases excluded from `audit_package` default share output
  · add `--include-imported` CLI opt-in flag · audit log still records run
  for forensics · M5 sets the manifest tag, audit_package filter is ~30 LOC
  follow-up.
- **D6**: M7 mesh budget two-tier · `mesh_mode="beginner"` (default) hard
  cap 5M · `mesh_mode="power"` cap 50M · single toggle inside Mesh Wizard.
- **D7**: M8 `beginner_report v0` = passive listing only (verdict warnings
  + case_quality deltas + identified sharp edges) · active anti-pattern
  detection deferred to M8.1 stretch.
- **D8**: 60-day window end gate = mandatory stranger dogfood — 1
  CFD-literate non-project-member runs end-to-end in target 30 min /
  cap 45 min · if blocked, the 60-day window is NOT done.

### Dogfood requirement (before claiming Done)

Real local upload of `examples/imports/ldc.stl` + `cylinder.stl` + 1
intentionally-broken non-watertight STL (honest reject). Mock-solver run
end-to-end (real OpenFOAM is M7). Append log to
`.planning/dogfood/M5_import_v0.md`.
