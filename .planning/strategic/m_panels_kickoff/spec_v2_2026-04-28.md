### A.4 Routing contract

```
/workbench/<case_id>                  → StepPanelShell (default step = 1)
/workbench/<case_id>?step=1           → StepPanelShell (Step 1)
/workbench/<case_id>?step=2           → StepPanelShell (Step 2)
/workbench/<case_id>/import (legacy)  → 301-style React redirect to /workbench/<case_id>?step=1
/workbench/<case_id>/edit (legacy)    → redirect to /workbench/<case_id>?step=2 (or step=3 once Setup lands)
/workbench/<case_id>/mesh (legacy)    → redirect to /workbench/<case_id>?step=2
/workbench/<case_id>/run (legacy)     → redirect to /workbench/<case_id>?step=4
```

Step state lives in URL (`?step=N`) so:
- back button works as expected
- deep links into a specific step work
- no cross-tree state-management library is needed

### A.5 Step definition (`StepDef`)

```ts
type StepId = 1 | 2 | 3 | 4 | 5;
type StepStatus = "pending" | "active" | "completed" | "error";
interface StepDef {
  id: StepId;
  shortLabel: string;       // e.g. "Import", "Mesh", "Setup", "Solve", "Results"
  longLabel: string;        // e.g. "1 · Geometry Import"
  status: StepStatus;       // derived from case state
  taskPanelComponent: React.ComponentType<StepTaskPanelProps>;
  viewportConfig: {
    format: "stl" | "glb" | "none";
    glbUrl: (caseId: string) => string | null;
    stlUrl: (caseId: string) => string | null;
  };
}
```

The 5-step config table is hardcoded in Tier-A. Engineer-customizable steps are explicitly out-of-scope.

### A.6 Three-button contract (Tier-A surface)

```ts
interface StepNavigationProps {
  onAiProcess: () => Promise<void> | null;  // null = AI 处理 hidden/disabled
  onPrevious: () => void;
  onNext: () => void;
  canAdvance: boolean;
  canRetreat: boolean;
  aiInFlight: boolean;
  aiErrorMessage?: string;
}
```

Tier-A wires `[AI 处理]` only on Step 2 (Mesh) — calls existing `/api/cases/<id>/mesh` to trigger gmsh+sHM. Other steps render the button in a disabled state with a tooltip ("Wires up in M-AI-COPILOT" / "Wires up in M7-redefined" / "Wires up in M-VIZ.results"). The button is **always present** so the contract is visible to the engineer; the wire-up is incremental.

`[下一步]` is **manually triggered**. It NEVER auto-fires on AI 处理 success — Addendum 3 §3.b binding constraint.

### A.7 Tailwind / design tokens

Reuse existing surface-{500..950} tokens from current pages. New tokens NOT introduced. The shell uses:
- Background: `bg-surface-950`
- Pane borders: `border-surface-800`
- Active step: `bg-accent-500/15 border-accent-500`
- Completed step: `text-accent-400 + check icon`
- Error step: `border-rose-500 text-rose-200`
- Pending step: `text-surface-500`

### A.8 vtk.js Viewport reuse

The center pane mounts a single `<Viewport>` (from M-VIZ) keyed on `(currentStepId, caseId)`. Step changes:
- Step 1: `format='glb'` + `glbUrl=/api/cases/<id>/geometry/render` (M-RENDER-API)
- Step 2: `format='glb'` + `glbUrl=/api/cases/<id>/mesh/render` (M-RENDER-API mesh wireframe)
- Steps 3-5: Tier-A renders Step 1's geometry as the placeholder background (so the panel doesn't go blank when the user clicks ahead of the wire-up)

The Viewport already disposes its kernel on unmount (M-VIZ Codex R4 #1) so step changes that swap the Viewport key are leak-free.

## §H · Failure modes / risks

| Risk | Mitigation |
|---|---|
| ImportPage extraction breaks the existing route in transit | Keep ImportPage as a thin route shell during the arc; switch the actual upload UI to `<Step1Import>` only when Step 4 of the implementation lands; legacy route renders the same component path. |
| Viewport key thrashing on step swaps (creates+destroys vtk.js context per click) | Verify the kernel-dispose path runs cleanly; if too slow, use a single Viewport instance + memoized glbUrl prop instead of a key-driven remount. |
| react-query absent on existing pages | Surface scan in Step 1 of Sequence; if absent, add it as part of the Step 2 skeleton commit (single PR scope, line-A only). |
| Step 2 mesh endpoint long-running | M-RENDER-API's `/mesh/render` is fast (<1s for fixture). Real-case sHM-meshed polyMesh may be larger; if >5s, add a loading indicator in the task panel (already in StepNavigation's `aiInFlight` state). |
| Bundle delta blows past +50 KB | Tier-A scoping deliberately avoids new component libraries; if the budget is hit, audit per-file imports and lazy-load Step 3-5 placeholders. |
| Codex pre-merge round count exceeds 5 (precedent: M-VIZ converged at R5) | If R5 still CHANGES_REQUIRED, halt and escalate to CFDJerry per RETRO-V61-001 §"halt at R5" pattern. |
