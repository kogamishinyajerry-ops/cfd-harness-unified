# DEC-V61-004: Land Path B C-class infra fixes C1 (comparator aliases) + C2 (parameter plumbing) + whitelist audit

## Decision summary

Following Kogami's dashboard screenshot showing 0 PASS / 2 HAZARD / 1 FAIL / 7 NO-RUN across the 10 golden V&V cases, the agent produced `docs/whitelist_audit.md` (342 lines) diagnosing three orthogonal bug classes:

- **A-class (metadata)** — whitelist.yaml metadata mislabels (e.g., Re=100 cylinder tagged `k-omega SST` when it should be `laminar`); 4 cases affected.
- **B-class (gold values)** — gold reference_values questionable against primary literature (e.g., Turbulent Flat Plate Re_x=25000 is laminar, so Spalding doesn't apply); 5 cases affected. **External gate required — DEC-V61-003 禁区.**
- **C-class (infra)** — comparator schema mismatches (×14) and parameter plumbing silent drift (×12) in the execution pipeline itself; 3 sub-items C1/C2/C3.

The audit concluded that B-class repairs on top of a still-silently-broken C-class pipeline would be re-broken, so C-class MUST land first.

This DEC records landing C1 + C2 (autonomous turf) as PR #4 on branch `feat/c-class-infra-fixes`. C3 (sampleDict auto-generation) is next PR; B-class gold remediation is external-gate.

### C1 — ResultComparator alias layer

- `src/result_comparator.py`: new `CANONICAL_ALIASES` table mapping canonical field names (e.g., `Nu_avg`) to their recorded aliases (e.g., `Nu_bar`, `Nusselt_avg`). New helper `_lookup_with_alias(dict, key)` tries canonical first, then aliases.
- Companion-axis lookup transparently resolves aliases.
- Failure `summary` now surfaces `tried_aliases` + `available_keys` for diagnostic clarity.
- Eliminates the `comparator_schema_mismatch ×14` re-occurrence path observed in dashboard FAIL/HAZARD verdicts.
- `tests/test_result_comparator.py`: +8 tests in `TestFieldAliases` covering canonical-preferred, alias-fallback, companion-axis alias, failure-summary exposure.

### C2 — Case-builder pre-run parameter-plumbing assertion

- `src/foam_agent_adapter.py`: new `ParameterPlumbingError` exception class.
- Two parsers: `_parse_dict_scalar(dict_text, key)` reads a scalar from an OpenFOAM dict file body; `_parse_g_magnitude(g_text)` reads gravity vector magnitude from `constant/g`.
- Two static verifiers:
  - `_verify_buoyant_case_plumbing(case_dir, task_spec)` reads back `constant/physicalProperties` + `constant/g`, recomputes `Ra_effective = (g·β·ΔT·L³)/(ν·α)`, compares to `task_spec.parameters.Ra` within 1% tolerance. Raise on drift.
  - `_verify_internal_channel_plumbing(case_dir, task_spec)` recomputes `Re_effective = (U·D_h)/ν`, compares to `task_spec.parameters.Re` within 1% tolerance.
- Called at the tail of `_generate_natural_convection_cavity` (wired for DHC + NC Cavity + Rayleigh-Bénard generator paths) and `_generate_steady_internal_channel` (Plane Channel + Fully Developed Pipe Flow).
- Eliminates the `parameter_plumbing_mismatch ×12` silent fallback path where a case was generated with default ν / β / ΔT despite the task spec requesting different values.
- `tests/test_foam_agent_adapter.py`: +14 tests split across `TestParameterPlumbingParsers`, `TestBuoyantPlumbingVerification`, `TestInternalChannelPlumbingVerification`. Covers: parser round-trip, tamper detection (physicalProperties edit → raise), g magnitude (9.81 vs 1.0), BL dimensions round-trip.
