# DEC-V61-001: Claude APP v6.1 Cutover — Sole Primary Driver landing

## Decision summary

Activate Model Routing v6.1 for cfd-harness-unified, superseding
v5.2 ADWM self-Gate autonomous-governance and v6.0 Codex-primary
/ Claude-co-primary pair.

Under v6.1:
- `claude-opus47-app` is Sole Primary Driver (holds Fix Plan, Phase,
  Track, Notion, local shell, and commit signoff).
- `codex-gpt54-xhigh` is demoted to Heterogeneous Code Tool, invoked
  only for `src/**`, `tests/**`, and `knowledge/gold_standards/**`
  (non-tolerance) touches. Codex no longer holds governance权威.
- Notion Gate retains only the 4 hard-floor守护者 duties: GS tolerance
  edits, project北极星 edits, Notion DB-level destruction, 主导-工具
  链路失调 (3-consecutive-Codex-failure).
- All governance-class decisions land in `.planning/decisions/` (and
  Notion Decisions DB when MCP is restored) with
  `autonomous_governance: true` + `claude_signoff: yes` required; the
  `codex_tool_invoked` flag + optional `codex_diff_hash` +
  `codex_tool_report_path` fields are mandatory fill.
- Commit trailer convention: `Execution-by: claude-opus47-app`
  (+ optional `· Codex-tool-diff: <hash>` for 禁区 touches,
   + optional `Gate-approve: <url>` for GS tolerance touches).

## Why

The v6.0 pair structure required joint Codex↔Claude signoff for every
governance-class decision, but the Claude APP (`com.anthropic.
claudefordesktop`) could not be reached via Computer Use, blocking
co-sign and starving the queue. S-003o closed with an acknowledged
"treat the package as Codex-signed and externally synced, not fully
co-signed" caveat — unsustainable for continued autonomous operation.

v6.1 dissolves the blocker by promoting the Claude APP itself to sole
primary. Codex's diff-production capability is retained via tool-use
semantics, preserving the异构对抗 principle for禁区 changes without
requiring synchronous joint sign.

## Alternatives considered

- **A1. Keep v6.0, fix Computer Use access to Claude APP.** Rejected:
  blocker is infrastructural and outside this project's scope;
  indefinite wait before governance queue moves.
- **A2. Return to pre-v5.1 external Notion Gate loop.** Rejected:
  manual paste-to-Notion is slow and Notion MCP is currently
  UNREACHABLE anyway; would stall all activity.
- **A3. Adopt v6.1 as specified by Kogami's takeover prompt.**
  CHOSEN.
