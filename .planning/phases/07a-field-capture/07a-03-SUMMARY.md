## Deviations From Plan (auto-fixed per Rule 1)

### [Rule 1 — Bug #1] OpenFOAM 10 sampledSets requires list-form, not dict-form

- **Found during:** Task 1 Step 3 (first integration run). Solver failed in 5.2s with `solver_success: false, duration 5.2s, deviations: []` and `[WARN] field artifact dir missing, skipping manifest`.
- **Root cause investigation:** ran minimal reproducible case in `cfd-openfoam` container (`/tmp/phase7a_realrun_probe.sh`) with Wave 1 functions block. Got:
  ```
  --> FOAM FATAL IO ERROR: Attempt to return dictionary entry as a primitive
  file: controlDict/functions/sample/sets from line 36 to line 40.
  in function Foam::functionObjects::sampledSets::read
  ```
  Verified OF 10 canonical template at `/opt/openfoam10/etc/caseDicts/postProcessing/graphs/graphUniform.cfg`:
  ```
  sets
  (
      line
      {
          type            lineUniform;   # NOT `uniform`
          ...
      }
  );
  ```
  Ran same probe with list-form + lineUniform — solver succeeded, postProcessing/sample/ populated.
- **Fix:** Changed `_emit_phase7a_function_objects` sample-block from dict-form `sets { uCenterline { type uniform; ... } }` to list-form `sets ( uCenterline { type lineUniform; ... } );`. Added defensive comment citing OF 10 template + the exact ITstream error.
- **Files modified:** `src/foam_agent_adapter.py` (~8 LOC)
- **Verification:** re-ran integration → 27.4s solver success → postProcessing + VTK + residuals all present.
- **Commit:** not yet (Wave 3 Task 3 will roll this into the atomic commit alongside DEC-V61-031).

### [Rule 1 — Bug #2] residuals.csv glob path incorrect — tar-extract basename collision

- **Found during:** Task 1 Step 4 (first successful solver run after Bug #1 fix). Found `residuals/0/residuals.dat` on disk but NO top-level `residuals.csv`.
- **Root cause:** Wave 1 code in `_capture_field_artifacts` globs `artifact_dir.glob("postProcessing/residuals/*/residuals.dat")` but `container.get_archive('{case}/postProcessing/residuals')` tar-extracts under **basename only** (`residuals/`), not the full container path. Same applies to `sample/` (but the sample extraction itself happens to work because we don't derive anything from it at this layer).
- **Fix:** Changed glob to `artifact_dir.glob("residuals/*/residuals.dat")`. Added defensive comment.
- **Files modified:** `src/foam_agent_adapter.py` (~1 LOC + comment)
- **Verification:** re-ran integration → `residuals.csv` present at artifact root with `Time,Ux,Uy,p` header.

### [Rule 4 — Architectural flag, NOT fixed] Same-filename collision across sample iterations

- **Found during:** Task 1 Step 6 HTTP response inspection.
- **Issue:** 3 `sample/{0,500,1000}/uCenterline.xy` files collapse to 3 manifest entries with identical `filename` and identical download `url`. `resolve_artifact_path` uses `rglob(filename)` and returns the first match — so downloading any of the 3 lists gets the same file regardless of iteration.
- **Why not auto-fix:** requires URL-schema decision (Rule 4):
  - Option 1: qualify URL with iteration subpath (`.../field-artifacts/sample/500/uCenterline.xy`) — breaks the "basename-only URL, traversal-hardened" invariant per Wave 2 decisions.
  - Option 2: rename on stage (e.g. `uCenterline_iter500.xy`) — breaks OF-native layout.
  - Option 3: accept MVP, only surface the latest iteration — loses historical data for Phase 7b render.
- **Recommendation:** include as an explicit question for Codex review (pre-merge). Do NOT fix in this wave.
- **Current state:** HTTP 200 works for 1 of 3 (latest wins); tests pass (they use a single-iter fixture).
