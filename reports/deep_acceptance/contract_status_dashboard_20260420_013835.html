Portfolio Lanes

按交付语义分组

如果你要把 10 个 case 带进 demo、scientific review 或治理讨论，最好先按这里的三条 lane 分流：哪些能当 physics-backed baseline，哪些可以展示但必须保留 caveat，哪些根本不该被包装成“只差一点”的问题。

Lane A · Physics-backed Demo
这些 case 可以作为对外 demo 或验收 anchor，但依旧应该引用 benchmark 与 source path。
  • 方腔顶盖驱动流 · 中心线 benchmark 与主涡核位置都已稳定，适合当作 physics-backed anchor。
  • 后台阶分离再附着 · 再附着长度与速度剖面已对齐，是另一条可信的内部流锚点。
Lane B · 可展示但必须带 caveat
这些 case 不是不能展示，而是展示时必须把 runtime hazard 或已知偏差一起抬到台面上。
  • 圆柱绕流卡门涡街 · Cd / Cl_rms / wake deficit 可信，但 Strouhal 仍带 canonical-band shortcut 风险。
  • 湍流平板边界层 · Cf 数值对了，但要继续把 Spalding fallback 是否触发暴露出来。
  • NACA0012 翼型外流 · 偏差集中在前缘 / 吸力峰，但方向已知、归因明确，属于可解释偏差。
  • Rayleigh-Benard 自然对流 · Ra=1e6 当前是 compatible，但这个结论不应被偷渡到高 Ra 腔体问题上。
Lane C · 治理 / 结构阻塞
这些 case 目前的主要问题不是“再调一点参数”，而是 physics contract / governance 语义还没闭环。
  • 全发展湍流管流 · 当前 adapter 还是矩形 duct，不是 circular pipe；这是物理类别错配，不是调参问题。
  • 差分加热方腔自然对流 · 测量链已到 Nu=77.82，但 gold=30.0 与文献区间冲突仍未裁决。
  • 平面通道流 DNS · 当前 laminar comparator patch 还没 fresh rerun，DNS wall-unit 物理面仍未真正闭环。
  • 轴对称冲击射流 · 报告里叫 Nusselt，但当前 extractor 实际比的是 Cf，属于术语-物理错位。
