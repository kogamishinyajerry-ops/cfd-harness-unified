## Why not re-run all 5?

Pragmatism. The 3 backfill DECs share these properties:
1. **Clear commit-message evidence of prior Codex rounds** (036 + 036c): text in commit subject/body explicitly names Codex finding IDs and round numbers. Either (a) Codex actually ran but the report wasn't archived (protocol nit, not correctness issue), or (b) DEC text was aspirational. Case (a) is the strongly more likely hypothesis.
2. **Surgical scope** (all 3): 036 is schema + verdict-engine edit; 036c is 3-file `_compare_vector` lookup widening; 039 is 4-file additive API.
3. **Already validated by downstream consumers**: 036's schema flip is consumed by 036b/038 threshold logic (both passed session-level testing); 036c's u+/y+ fix enables plane_channel gate semantics; 039's `profile_verdict` is rendered in LDC UI and visually inspected.
4. **Codex account budget preservation**: current active account at 70% remaining; running 5 parallel reviews risks quota burn on next Phase 8 work.

Re-running 036b + 038 (high-risk new ~1100 LOC CFD modules) captures the highest-value independent signal. Backfill-trusting 036 + 036c + 039 (lower-LOC, already-referenced-downstream changes) is defensible judgment given the evidence.

## Honest risk disclosure

This backfill decision carries 3 residual risks:

1. **Protocol drift**: RETRO-V61-001's "≤0.70 → pre-merge" rule was designed to prevent self-overconfidence. Backfilling without fresh Codex run on 036 (0.65) and 036c (0.70) technically bends the rule. If a future regression surfaces in G1 or G2 logic, this note should be cited as partial cause.
2. **Ghost-Codex hypothesis unproven**: "Commits reference Codex rounds" is circumstantial. No transcript log captures the actual Codex feedback on 036 rounds 1+2 or 036c nit. If those rounds didn't actually happen as described, we're approving un-reviewed code.
3. **036c self-pass=0.70 is the literal boundary**: the `≤0.70` phrasing includes 0.70. Reading strictly, 036c requires pre-merge. Backfill overrides by judgment.

## Mitigation

- If any of 036/036c/039 logic fires spuriously or misses a regression in Phase 8 Sprint 2 testing, this note auto-promotes to "Codex re-run required" status.
- Any future DEC with `self-pass ≤0.70` must re-run Codex regardless of commit-message evidence.
- Next retrospective (RETRO-V61-004 at counter=40) should review whether ghost-Codex situations recurred; if yes, protocol update to require artifact archival at round time.
