§3为什么 FUSE 而不跑 cycle 2？

ADWM v5.2 的 2-cycle budget 允许 cycle 2 用 Option B (face-integrated snGrad(T)) 替换 between-first-two-cells gradient。理论上 Option B 是更精确的 wall heat flux 近似。

问题是：on a well-posed physical setup，Option B 应该给出 ≥ Option C 的 mean，因为 wall-adjacent gradient 本身是 downward-biased proxy。所以 Option B 预期落在 ≥ 78，mesh 继续 refine 时可能逼近 140-160。

Cycle 2 会烧掉 Codex 额度和 solver wall-clock 去确认一个理论上已知的方向——而且离 gold=30 越来越远。 这既不是收敛 bracket，也不是关闭真正的开放问题。

真正的开放问题是一个 gold-reference 准确性问题，只有外部 Gate（或 Kogami 本人）能裁决。

§4三个文献源的交叉验证

三个独立文献源都指向 Ra=1e10 2D DHC 的真实 Numean 在 100-160 区间，而非 gold 记录的 30。这是 FUSE 的物理依据。

§5外部 Gate 可选的两条路径

Path P-1

Gold Correction
