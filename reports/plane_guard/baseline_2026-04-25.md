## Test pollution remediation (RETRO-V61-006 MP-D)

The first baseline run produced 1 line in `sys_modules_pollution.jsonl` and (after subsequent investigation) 2 lines in `fixture_frame_confusion.jsonl` — all from tests that legitimately exercise the plane-guard bypass paths. Specifically:

- `tests/test_plane_guard_observability.py::test_atexit_pollution_check_runs_diff_when_snapshot_active` deliberately polluted `src.task_runner` to verify A13 fires
- `tests/test_plane_guard.py::test_a3c_test_allowlist_permits_forbidden_pair` (and a parametrize sibling) exercises the test-allowlist bypass on `src.foam_agent_adapter → import src.result_comparator`, which IS the canonical A18 fixture-frame confusion incident

These produced ~2 incidents per CI run — enough to trigger the §2.4 rollback evaluator (threshold 3 in 14 days) after just 2 CI runs from test pollution alone. Net effect: signal-to-noise ratio of the dogfood window would be unrecoverable.

**Fix landed**: extended `tests/conftest.py::_plane_guard_test_isolation` autouse session fixture to redirect `src._plane_guard._find_repo_root` to a session-scope `tmp_path_factory.mktemp(...)` for the entire test session. Per-test `monkeypatch.setattr(guard_module, "_find_repo_root", ...)` in `tests/test_plane_guard_observability.py` provides defense-in-depth.

After the fix, both .jsonl files remain uncreated through a clean session run (above baseline). Real production-path A13 / A18 signals (from `src/__init__.py` auto-install in user code, NOT test code) will continue to write to repo-root anchored paths as designed.
